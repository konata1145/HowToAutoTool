테스트 자동화 도구의 필요성

1. Why we need web testing?

웹의 목적은 다양한 브라우저와 운영 체제를 준수하며 사용자와 효율적으로 정보를 주고받는 것이다

웹 테스트는 요구 사항에 따라 품질과 기능을 보장하는 소프트웨어 테스트의 관행이다

기능 일치, 보안 위반, 통합 문제, 환경 문제 등과 같은 문제들을 탐지하기 위해 필요하다

2. Web Testing Types

기능 테스트 - 가장 일반적인 테스트 중 하나로 빌드가 설계에 맞게 작동하는지 확인한다. 링크 테스트, 쿠키 테스트, HTML 및 CSS 검증, 데이터베이스 연결 등을 파악한다

인터페이스 테스트 - 웹 인터페이스가 인터럽트에 어떻게 반응하는지, 그리고 다른 서버와의 호환성 및 상호 작용을 검사한다

호환성 테스트 - 웹 설계가 다양한 브라우저나 장치와 호환이 되는지 여부를 확인한다

성능 테스트 - 트래픽과 확장성을 테스트 할 수 있다

3. Efficient Testing Methods

브라우저 간 호환성 테스트를 엄격하게 수행해야 한다

다양한 조건에서 성능을 테스트 해야 한다

부하 테스트가 점진적으로 수행되는지 확인해야 한다

보안 테스트 시 URL 문자열을 변경할 수 없도록 유지해야 한다
